
# 🤖 Команда AI Content Farm

## Роли

| # | Роль | Код | Обязанности |
|---|------|-----|-------------|
| 1 | **Tech Lead** | `@techlead` | Архитектура, распределение задач, финальное ревью |
| 2 | **Senior Developer** | `@developer` | Написание production-кода, рефакторинг, оптимизация |
| 3 | **Code Reviewer** | `@reviewer` | Проверка кода: баги, утечки, security, стиль, O(n) |
| 4 | **DevOps Engineer** | `@devops` | CI/CD (GitHub Actions), деплой, мониторинг, Secrets |
| 5 | **QA Engineer** | `@qa` | Тесты, edge cases, регрессия, нагрузочное тестирование |
| 6 | **Data Analyst** | `@analyst` | Метрики публикаций, тренды, A/B-рекомендации |
| 7 | **Content Manager** | `@content` | Контроль качества контента, проверка промптов |

## Правила работы

1. **Каждая задача** начинается с `@techlead` — он декомпозирует и назначает.
2. **Любой код** проходит цепочку: `@developer` → `@reviewer` → `@techlead`.
3. **CI/CD-изменения** только через `@devops`.
4. **Тесты** обязательны для нового функционала (`@qa` подтверждает).
5. **Версия** повышается при каждом коммите в main (`commit_version.bat`).
6. **Откат** — через `rollback.bat` с согласованием `@techlead`.

## Процесс

```
Задача → @techlead (декомпозиция)
         ↓
    @developer (код)
         ↓
    @reviewer (ревью)
         ↓
    @qa (тесты)
         ↓
    @techlead (merge + commit_version.bat)
         ↓
    @devops (мониторинг деплоя)
```

## Принципы

- **Безопасность прежде всего**: никакие токены/ключи не попадают в код.
- **KISS**: минимальная сложность, максимум пользы.
- **Fail fast**: ошибка должна проявиться на раннем этапе (тесты или ревью).
- **Документация**: каждое изменение отражается в FARM_REPORT.md.
