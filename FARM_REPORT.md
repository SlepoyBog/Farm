## 5. Платформы и доход

### Telegram
- Канал: публикация с HTML-форматированием
- Адаптация под TG-стиль: короткие абзацы, эмодзи, вовлекающие вопросы
- Лимит: 4096 символов на сообщение
- **Картинка**: `sendMessage` + `link_preview_options.url` (preview внизу, ссылки в тексте нет)
- ~~Авто-кросс-постинг в Яндекс.Дзен~~ — Дзен не видит фото в `sendMessage`. **Решение**: настроить Дзен на RSS сайта.

### VK
- Адаптация на лету: HTML → текст + хештеги + вопрос
- Поддержка digest-постов (несколько тем в одном посте)
- API VK v5.199

### OK.ru (Одноклассники)
- Модуль: `src/ok_publisher.py` — API `ok.wall.post` с подписью запросов
- Адаптация: HTML → текст (до 8000 символов) + хештеги
- Фото: загрузка через `photos.getUploadUrl` + прикрепление к посту
- Подпись запросов: `md5(params + session_secret)`, OAuth 2.0
- Настройка: регистрация приложения на `apiok.ru/dev`, токен с правами `GROUP_CONTENT`, `PHOTO_CONTENT`

### SEO-сайт (GitHub Pages)
- Адрес: `SITE_URL` из `.env` (сейчас https://your-niche-blog.pages.dev)
- Генерация: HTML-страницы + sitemap.xml + RSS + robots.txt
- Deploy: `deploy.bat` → git push → GitHub Actions → Pages
- SEO-теги: title, description, keywords, Open Graph, canonical
- Статей на сайте: **116**

## 8. Feedback Loop

Собирает метрики из VK (просмотры, лайки, комментарии, репосты).  
Анализирует топ-3 поста за 30 дней.  
Генерирует рекомендации по стилю и структуре.  
Рекомендации впрыскиваются в system_prompt при генерации новых статей.

**Текущее состояние:** есть история 40+ постов, метрики VK собираются.  
Telegram-метрики не собираются (Bot API не отдаёт статистику каналов).

## 9. Известные ограничения

- Google Trends (`pytrends`) может не работать без авторизации — запасной вариант: post_history + сезонная ротация
- Telegram: метрики (просмотры/реакции) не собираются из-за ограничений Bot API
- GitHub Pages: нет серверной логики, только статика
- VK digest (комбинированный пост): собирается из базовых HTML-статей, без адаптации (может выглядеть неидеально)
- **VK фото**: `photos.getWallUploadServer` требует user-токен (не group). Сейчас фото не прикрепляются к постам VK до получения правильного токена.
- **Яндекс.Дзен**: забирает посты только из `sendPhoto` (photo + caption ≤ 1024). `sendMessage` с `link_preview_options` не распознаётся как пост с фото. **Решение**: настроить Дзен на парсинг RSS сайта (`site/rss.xml`).

## 10. Тестовый режим

Ферма запускается в **тестовом режиме** — 1 пост за запуск (вместо 15).  
Это позволяет отладить баги без лишних расходов и спама.

| Режим | Команда | Постов |
|-------|---------|:------:|
| Тестовый (по умолчанию) | `python src/main.py` | 1 |
| Полный | `python src/main.py --full` / `run_prod.bat` | 15 |

Дополнительно:
- Записи о публикациях всё равно попадают в `post_history`
- Когда ферма будет полностью отлажена, тестовый режим отключается

| `UNSPLASH_ACCESS_KEY` | API-ключ Unsplash (бесплатный, регистрация на unsplash.com) |
| `NEWSAPI_KEY` | API-ключ NewsAPI.org (опционально, 100 запр/день бесплатно) |
| `OK_ACCESS_TOKEN` | OAuth-токен Одноклассники API |
| `OK_PUBLIC_KEY` | Public key приложения OK |
| `OK_APP_SECRET` | Secret key приложения OK |
| `OK_GROUP_ID` | ID группы OK (числовой) |

### Процесс

1. GitHub Actions запускает `python src/main.py` (1 пост, тестовый режим)
2. Запускает `python src/feedback_loop.py` (сбор метрик)
3. Коммитит и пушит `site/` обратно в репозиторий (с `[skip ci]` для предотвращения цикла)

Переменные окружения передаются через `${{ secrets.* }}` — `load_dotenv()` в коде не перезаписывает уже установленные env-переменные, поэтому изменений в коде не требуется.

## 13. Версионирование и откаты

### Система версий

Каждая завершённая сессия (локальная или через GitHub Actions) фиксируется с порядковым номером версии.

- **Файл `VERSION`** в корне проекта содержит текущий номер (целое число).
- **Коммиты** имеют формат: `v<N>: <описание> [skip ci]`
- Тег `[skip ci]` предотвращает цикл GitHub Actions (коммит → триггер → коммит → ...)

### Rollback (откат)

Скрипт `rollback.bat` в корне проекта:

| Команда | Действие |
|---------|----------|
| `rollback.bat` | Показать справку и список версий |
| `rollback.bat last` | Откат на 1 коммит назад (soft reset) |
| `rollback.bat <N>` | Откат к версии vN (soft reset) |

После отката изменения остаются в индексе — нужно сделать новый коммит:
```
git commit -m "v<N>: rollback to v<N> [skip ci]"
git push origin main
```
